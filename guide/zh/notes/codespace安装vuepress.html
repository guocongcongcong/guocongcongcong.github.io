# 如何在codespace部署一个vuepress

  • 对codespace了解不深，看到了想用一下，没看文档，没做攻略，直接进来就开始用，摸着石头过河，遇到了很多问题。
  • 前端页面类似于vscode,现在目测插件也可以相同的。

# 感受

  • 在codespace内写播客，就不再需要固定的电脑了，只有有一个浏览器，能登陆github就可以随时开始。甚至可以实时同步
  • 但是个人感觉，其实他的功能远不止于此，还有待我大力发掘。
  • vuepress是很好的博客框架，第一次接触就很快能上手，虽然还有很多需要学习，但是整体整体感受很好，而且我习惯了写markdown，这个用起来就很舒服。
  • 这次把他俩结合起来，算是意外收获。
  • 先这样，之后想起来什么再说。

# 环境

  • codespace类似于一个docker容器或者linux虚拟机。单个空间是一个单独的虚拟机。它的生命周期为从创建到删除。中间关闭，停止，其中的数据都不会被删除。
  • 我使用的是空白的template,该机器内部，直接内置了node(v19.8.1)，npm(9.5.1)，yarn(1.22.19),java(openjdk version "17.0.6" 2023-01-17 LTS),python(Python 3.10.4),nvm(0.39.2),curl。我个人只坚持了这些东西，其他的如果有需要可以自行确认。

# 安装

  • 在知道了基本的node环境之后，就可以进行安装了。

整体创建参考：https://cchroot.github.io/2020/06/20/vuepress%E5%BF%AB%E9%80%9F%E5%88%B6%E4%BD%9Cvue%E6%96%87%E6%A1%A3%E5%8D%9A%E5%AE%A2/ (opens new window)

  • 但是全抄可能会遇到一些问题，主要在于权限和配置问题，在这里PO出我的package.json，如果能减少您踩坑，我会很高兴。

    • 注意codespace里的主线名称为：main，可能存在不存在master分支的情况，

# 一键部署

  • 如果你好好看了以上的文档，应该不会遇到其他什么大的问题，可以直接在控制台输入以下命令yarn deploy，就可以一键部署到github pages上了。一般地址为：https://<username>.github.io/

# 附录

  • deploy.sh
# 确保脚本抛出遇到的错误
set -e

# 打包生成静态文件
yarn  docs:build

# 进入打包好的文件夹
cd docs/.vuepress/dist

# 创建git的本地仓库，提交修改
git init
git add -A
git commit -m 'deploy'

# 覆盖式地将本地仓库发布至github，因为发布不需要保留历史记录
# 格式为：git push -f git@github.com:'用户名'/'仓库名'.git master
git push -f git@github.com:guocongcongcong/guocongcongcong.github.io.git main

cd -
